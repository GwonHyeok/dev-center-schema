1N Region → Center / Review · User → Review

하나의 지역은 여러 센터를 포함, 하나의 센터는 여러 후기를 가짐. 한 유저는 여러 후기 작성.

N:M Center ↔ Tag / Review ↔ Tag

한 센터/후기는 여러 태그를 가지고, 한 태그는 여러 센터/후기에 붙음. 조인 테이블(center_tag, review_tag) 경유.

1N Review → ReviewReport

하나의 후기에 여러 신고가 쌓임. 신고 사유 별도 기록해 관리자 검토 시 패턴 분석 가능.

N1 AuditLog (polymorphic)

모든 엔티티 변경을 하나의 로그 테이블로 통합 기록. entity 문자열 + entity_id로 다형적(polymorphic) 참조.

Q1 답변

의뢰하신 9가지 기능 → 데이터 설계 매핑

의뢰서 원문을 그대로 가져와, 각 기능이 어떤 테이블·필드·관계로 구현되는지 1:1로 연결했습니다.

의뢰 요구사항
"지역 기반 센터 리스트 조회"
설계 방향
지역을 서울 → 양천구 → 목동 같은 계층 구조로 저장합니다. 구를 선택하면 하위 동의 센터까지 자동 포함해서 보여줍니다.
의뢰 요구사항
"센터명 및 치료 유형 기반 검색"
설계 방향
센터 이름으로 빠르게 찾는 검색 + 치료 유형(언어·감각통합·놀이) 필터를 함께 제공. 데이터가 수만 건으로 늘어나도 즉시 결과가 나오도록 검색용 색인을 미리 만들어둡니다.
의뢰 요구사항
"센터 상세 페이지 (후기 요약, 후기 미리보기)"
설계 방향
센터마다 '후기 개수 · 인기 태그 3개'를 미리 계산해 저장해 둡니다. 상세 페이지가 열릴 때 매번 계산하지 않고 바로 불러오기만 하므로 후기가 수천 개 쌓여도 느려지지 않습니다.
의뢰 요구사항
"후기 리스트 조회 (최신순 정렬)"
설계 방향
스크롤을 내리면 이어서 보이는 무한 스크롤 방식. 오래된 후기가 쌓여도 앞부분 로딩 속도는 처음과 동일하게 유지됩니다.
의뢰 요구사항
"후기 작성 기능 (태그 선택 + 한 줄 후기)"
설계 방향
글은 한 줄(100자 이내), 태그는 카테고리별로 '치료 유형 1개 + 강점 여러 개' 같은 선택 규칙으로 구성. 학부모가 부담 없이 빠르게 쓸 수 있도록 진입장벽을 낮춥니다.
의뢰 요구사항
"카카오 로그인 (후기 작성 시)"
설계 방향
탐색·리스트·상세는 로그인 없이 자유롭게. 후기 작성 버튼을 누르는 순간에만 카카오 로그인을 요구하는 '선택적 로그인' 방식으로 학부모 이탈을 최소화합니다.
의뢰 요구사항
"카카오톡 공유 기능"
설계 방향
카카오톡으로 센터를 공유하면 받은 사람이 바로 센터 상세를 열 수 있게 공유 카드에 센터명·지역·대표 태그를 넣습니다.
의뢰 요구사항
"네이버 지도 연동 (외부 이동)"
설계 방향
센터 상세의 '지도 보기' 버튼을 누르면 네이버 지도 앱/웹으로 이동. 좌표도 함께 저장해둬서 나중에 앱 안에서 직접 지도 보기로 확장할 때 준비가 되어 있습니다.
의뢰 요구사항 (핵심)
"관리자 페이지 (센터/후기/태그/지역 관리)"
설계 방향
4개 항목(센터·후기·태그·지역)을 각각 관리하는 페이지 + 아래 6가지 운영 기능. 누가·언제·뭘 수정했는지 자동 기록되어 언제든 복원 가능합니다.
관리자 페이지 — 운영자가 실제로 하는 6가지
센터
CSV 일괄 업로드
초기 데이터를 엑셀 파일로 한 번에 등록. 1개 시범 지역 센터 수십 곳 시드 작업에 필수.
지역
트리 뷰 + 누락 표시
지역을 계층으로 보고, 센터 0개인 지역을 별색으로 표시해 확장 우선순위 파악.
태그
동의어 통합
'언어치료/언어발달' 같은 중복 태그는 aliases 배열로 묶어 검색 통합.
후기
신고 검토 워크플로우
신고 누적 후기는 목록 상단에 뜸. 숨김/복원 클릭 한 번. 신고 사유 통계로 악용 감지.
감사
변경 이력 자동 기록
운영자 복수여도 누가 언제 무엇을 고쳤는지 audit_log 자동 기록. 복원 가능.
통계
기본 대시보드
지역별 센터·후기 수, 태그 인기 순위, 신규 가입자 추이 — 운영 감을 잡을 최소 지표.

확장 ①
복합 인덱스 설계

'지역 + 치료유형' 같은 자주 쓰는 조건을 복합 인덱스로 묶어 수십만 건 데이터에서도 일정한 조회 속도.

확장 ②
검색 업그레이드 어댑터

초기엔 PostgreSQL full-text → 데이터가 수만 건을 넘으면 Meilisearch/Algolia로 교체. 앱 코드는 그대로.

확장 ③
RLS로 2차 기능 대비

'센터가 자기 후기만 보는' B2B 페이지 추가 시, Supabase Row Level Security 정책 추가만으로 대응.
