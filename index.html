Q1 답변

의뢰하신 9가지 기능 → 데이터 설계 매핑

의뢰서 원문을 그대로 가져와, 각 기능이 어떤 테이블·필드·관계로 구현되는지 1:1로 연결했습니다.

의뢰 요구사항
"지역 기반 센터 리스트 조회"
설계
Region 테이블을 parent_id 자기참조로 계층화 (도 → 시 → 구 → 동). Center는 region_id로 소속. 구 단위 선택 시 하위 동까지 포함해 조회.
의뢰 요구사항
"센터명 및 치료 유형 기반 검색"
설계
센터명은 PostgreSQL full-text 인덱스. 치료 유형은 Tag(category='type') 필터로, center_tag 조인으로 검색. 복합 인덱스 (region_id, status)로 리스트 조회 성능 확보.
의뢰 요구사항
"센터 상세 페이지 (후기 요약, 후기 미리보기)"
설계
Center에 캐시 필드 (review_count, top_tags) 추가. 후기 작성·삭제 시 트리거로 자동 갱신 → 상세 페이지 로딩 1회 쿼리로 완료. 미리보기는 최신 3개를 join으로 같이 내림.
의뢰 요구사항
"후기 리스트 조회 (최신순 정렬)"
설계
Review에 복합 인덱스 (center_id, status, created_at DESC). 무한 스크롤은 created_at 커서 기반 페이지네이션으로 수만 건도 일정 속도 유지.
의뢰 요구사항
"후기 작성 기능 (태그 선택 + 한 줄 후기)"
설계
Review에 한 줄 제한 (CHECK length(content) ≤ 100). 태그는 review_tag 다대다. 태그는 카테고리별로 그룹핑돼 "치료 유형 1개 + 강점 여러 개" 같은 선택 규칙 적용.
의뢰 요구사항
"카카오 로그인 (후기 작성 시)"
설계
Supabase Auth + Kakao OAuth Provider 연동. 탐색·리스트·상세는 비로그인 허용, 후기 작성 버튼 클릭 시에만 로그인 요구하는 '선택적 인증'으로 학부모 진입장벽 최소화.
의뢰 요구사항
"카카오톡 공유 기능"
설계
Kakao JavaScript SDK로 피드형 공유. 공유 카드에는 센터명, 지역, 대표 태그 3개, 공유 전용 딥링크를 넣어 앱으로 바로 복귀하도록 구성. 스키마에 별도 테이블 불필요.
의뢰 요구사항
"네이버 지도 연동 (외부 이동)"
설계
Center 테이블에 naver_map_url 필드 저장. 상세 화면 '지도 보기' 버튼 → 네이버 지도 앱/웹으로 이동(fallback 처리). 좌표(lat/lng)도 함께 저장해 추후 인앱 지도 전환에 대비.
의뢰 요구사항 (핵심)
"관리자 페이지 (센터/후기/태그/지역 관리)"
설계
4개 엔티티 각각 CRUD + 아래 6가지 운영 기능 내장. "누가·언제·뭘 수정했는지"는 audit_log 테이블에 자동 기록 → 복원 가능.
관리자 페이지 — 운영자가 실제로 하는 6가지
센터
CSV 일괄 업로드
초기 데이터를 엑셀 파일로 한 번에 등록. 1개 시범 지역 센터 수십 곳 시드 작업에 필수.
지역
트리 뷰 + 누락 표시
지역을 계층으로 보고, 센터 0개인 지역을 별색으로 표시해 확장 우선순위 파악.
태그
동의어 통합
'언어치료/언어발달' 같은 중복 태그는 aliases 배열로 묶어 검색 통합.
후기
신고 검토 워크플로우
신고 누적 후기는 목록 상단에 뜸. 숨김/복원 클릭 한 번. 신고 사유 통계로 악용 감지.
감사
변경 이력 자동 기록
운영자 복수여도 누가 언제 무엇을 고쳤는지 audit_log 자동 기록. 복원 가능.
통계
기본 대시보드
지역별 센터·후기 수, 태그 인기 순위, 신규 가입자 추이 — 운영 감을 잡을 최소 지표.

확장 ①
복합 인덱스 설계

'지역 + 치료유형' 같은 자주 쓰는 조건을 복합 인덱스로 묶어 수십만 건 데이터에서도 일정한 조회 속도.

확장 ②
검색 업그레이드 어댑터

초기엔 PostgreSQL full-text → 데이터가 수만 건을 넘으면 Meilisearch/Algolia로 교체. 앱 코드는 그대로.

확장 ③
RLS로 2차 기능 대비

'센터가 자기 후기만 보는' B2B 페이지 추가 시, Supabase Row Level Security 정책 추가만으로 대응.

부록 · 실제 SQL 스키마

의뢰 기능이 모두 반영된 스키마

-- 1. 지역 (계층 구조) — "지역 기반 센터 리스트 조회" CREATE TABLE region ( id bigserial PRIMARY KEY, name text NOT NULL, parent_id bigint REFERENCES region(id), level smallint NOT NULL, sort_order int DEFAULT 0 ); -- 2. 발달센터 — "센터 상세 (후기 요약, 미리보기)" 대응 캐시 필드 포함 CREATE TABLE center ( id bigserial PRIMARY KEY, name text NOT NULL, region_id bigint NOT NULL REFERENCES region(id), address text, phone text, naver_map_url text, lat double precision, lng double precision, review_count int DEFAULT 0, -- 후기 요약 캐시 top_tags text[] DEFAULT '{}', -- 상위 태그 캐시 status text DEFAULT 'active', created_at timestamptz DEFAULT now(), updated_at timestamptz DEFAULT now() ); CREATE INDEX ON center (region_id, status); -- 3. 태그 (카테고리화) — "태그 선택 + 한 줄 후기" 대응 CREATE TABLE tag ( id bigserial PRIMARY KEY, name text NOT NULL UNIQUE, category text NOT NULL, -- type | strength | env aliases text[] DEFAULT '{}', parent_id bigint REFERENCES tag(id), sort_order int DEFAULT 0 ); -- 4. 후기 — "후기 리스트 (최신순 정렬)" 대응 인덱스 CREATE TABLE review ( id bigserial PRIMARY KEY, center_id bigint NOT NULL REFERENCES center(id), author_id uuid NOT NULL REFERENCES auth.users(id), -- 카카오 로그인 유저 content text NOT NULL CHECK (length(content) <= 100), -- 한 줄 status text DEFAULT 'visible', report_count int DEFAULT 0, created_at timestamptz DEFAULT now() ); CREATE INDEX ON review (center_id, status, created_at DESC); -- 5. 조인 테이블 (다대다) CREATE TABLE center_tag ( center_id bigint REFERENCES center(id) ON DELETE CASCADE, tag_id bigint REFERENCES tag(id), PRIMARY KEY (center_id, tag_id) ); CREATE TABLE review_tag ( review_id bigint REFERENCES review(id) ON DELETE CASCADE, tag_id bigint REFERENCES tag(id), PRIMARY KEY (review_id, tag_id) ); -- 6. 신고 상세 — 관리자 검토용 CREATE TABLE review_report ( id bigserial PRIMARY KEY, review_id bigint NOT NULL REFERENCES review(id), reporter_id uuid NOT NULL, reason text NOT NULL, resolved boolean DEFAULT false, created_at timestamptz DEFAULT now() ); -- 7. 감사 로그 — "누가 언제 뭘 수정했는지 자동 기록" CREATE TABLE audit_log ( id bigserial PRIMARY KEY, actor_id uuid NOT NULL, entity text NOT NULL, entity_id bigint NOT NULL, action text NOT NULL, diff jsonb, created_at timestamptz DEFAULT now() );
