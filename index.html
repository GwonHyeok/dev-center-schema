센터 · 후기 · 태그 · 지역 데이터가 계속 누적되고, 운영자가 관리자 페이지에서 직접 관리하는 구조를 4개 핵심 엔티티와 워크플로우 기반으로 설계했습니다. 비개발자도 이해 가능한 구조 설명부터, 실제 SQL 스키마까지 한 번에 보여드립니다.

01 · 데이터 모델

4개 엔티티가 서랍처럼 분리되어 연결됩니다

📍
Region
지역
  • PK id
  • name (양천구)
  • parent_id → self
  • level (도/시/구/동)
  • sort_order
Tree 예시
서울특별시
├─ 양천구
│ └─ 목동 · 신정동
└─ 강남구
🏥
Center
발달센터
  • PK id
  • name
  • region_id → Region
  • address
  • phone
  • naver_map_url
  • status (active/hidden)
  • created_at
관계
↑ Region 다대일 ↔ Tag 다대다
🏷️
Tag
태그
  • PK id
  • name (언어치료)
  • category (유형/강점/환경)
  • aliases[] (동의어)
  • parent_id → self
  • sort_order
카테고리 예
[유형] 언어치료 · 감각통합 · 놀이치료
[강점] 친절한 선생님 · 깔끔한 시설
[환경] 주차 편리 · 대중교통 가까움

Review
후기
  • PK id
  • center_id → Center
  • author_id → User
  • content (한줄)
  • status (visible/hidden)
  • report_count
  • created_at
관계
↑ Center 다대일 ↔ Tag 다대다
center_tag 조인 테이블
center_id · tag_id · (type: 제공/특징)
review_tag 조인 테이블
review_id · tag_id
왜 분리했나요? 각 서랍은 서로 '연결'만 되어 있어서, 지역을 늘리거나 태그를 새로 만들어도 기존 데이터를 건드리지 않고 확장됩니다. 예시) 새 동이 생기면 Region 트리에 한 줄만 추가 → 기존 Center·Review 데이터는 그대로 유지.
02 · 관리자 설계

운영자가 페이지를 열면 바로 할 수 있는 것

1
센터 추가
  • • 화면에서 한 건씩 직접 입력
  • • 엑셀(CSV) 파일로 수십 개 한 번에 업로드
  • • 누락된 지역은 자동 리스트업
2
지역 관리
  • • 트리 뷰로 한눈에 보기
  • • 새 동·구 생기면 한 줄 추가
  • • 센터가 없는 지역은 표시로 구분
3
태그 관리
  • • 카테고리(유형/강점/환경)별 그룹화
  • • '언어치료 = 언어발달' 같은 동의어 통합
  • • 자주 쓰이는 태그 정렬 우선순위
4
후기 검토
  • • 신고된 후기 목록에서 검토
  • • 숨김 / 복원 클릭 한 번
  • • 신고 사유 통계로 악용 감지
5
감사 로그 (자동)

누가 · 언제 · 무엇을 수정했는지 자동 기록됩니다. 문제가 생기면 이전 상태로 복원 가능. 운영자 여러 명이 될 때도 책임 추적 가능한 구조.

03 · 확장 고려

데이터가 쌓여도 느려지지 않고, 2차 기능이 자연스럽게 붙는 구조

성능
복합 인덱스 설계

'지역 + 치료 유형' 같은 자주 쓰는 조건을 복합 인덱스로 묶어 수십만 개 데이터도 빠르게 조회. 후기 수가 쌓여도 리스트 로딩 시간 일정 유지.

검색 업그레이드
어댑터 교체 가능

초기엔 PostgreSQL 자체 검색 기능으로 충분. 데이터가 수만 개를 넘어서면 Meilisearch / Algolia로 검색부만 교체. 앱 코드는 그대로 유지.

2차 확장
권한 정책 사전 준비

2차에서 '센터가 직접 로그인해서 자기 후기만 보는 페이지'가 생길 때, Supabase Row Level Security로 정책 추가만 하면 됨. 스키마 재설계 불필요.

04 · 실제 SQL 스키마 (요약)

개발자 관점에서도 바로 쓸 수 있는 형태

-- 지역 (계층 구조) CREATE TABLE region ( id bigserial PRIMARY KEY, name text NOT NULL, parent_id bigint REFERENCES region(id), level smallint NOT NULL, -- 1:도, 2:시, 3:구, 4:동 sort_order int DEFAULT 0, created_at timestamptz DEFAULT now() ); -- 발달센터 CREATE TABLE center ( id bigserial PRIMARY KEY, name text NOT NULL, region_id bigint NOT NULL REFERENCES region(id), address text, phone text, naver_map_url text, status text DEFAULT 'active', -- active | hidden created_at timestamptz DEFAULT now(), updated_at timestamptz DEFAULT now() ); CREATE INDEX ON center (region_id, status); -- 태그 CREATE TABLE tag ( id bigserial PRIMARY KEY, name text NOT NULL UNIQUE, category text NOT NULL, -- type | strength | env aliases text[] DEFAULT '{}', parent_id bigint REFERENCES tag(id), sort_order int DEFAULT 0 ); -- 후기 CREATE TABLE review ( id bigserial PRIMARY KEY, center_id bigint NOT NULL REFERENCES center(id), author_id uuid NOT NULL REFERENCES auth.users(id), content text NOT NULL CHECK (length(content) <= 100), status text DEFAULT 'visible', -- visible | hidden report_count int DEFAULT 0, created_at timestamptz DEFAULT now() ); CREATE INDEX ON review (center_id, status, created_at DESC); -- 조인 테이블 (다대다) CREATE TABLE center_tag ( center_id bigint REFERENCES center(id) ON DELETE CASCADE, tag_id bigint REFERENCES tag(id), PRIMARY KEY (center_id, tag_id) ); CREATE TABLE review_tag ( review_id bigint REFERENCES review(id) ON DELETE CASCADE, tag_id bigint REFERENCES tag(id), PRIMARY KEY (review_id, tag_id) ); -- 감사 로그 (운영자 변경 추적) CREATE TABLE audit_log ( id bigserial PRIMARY KEY, actor_id uuid NOT NULL, entity text NOT NULL, -- center | review | tag | region entity_id bigint NOT NULL, action text NOT NULL, -- create | update | delete | hide diff jsonb, created_at timestamptz DEFAULT now() );
